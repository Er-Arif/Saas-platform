# Company Platform Architecture

This repository is the parent company platform for a software business that sells:

- downloadable software products
- hosted services and APIs
- future enterprise deployments

Core principle:

- Platform auth is only for company commercial relationships.
- Product auth is decentralized and independent.
- All assets belong to an organization.

Primary surfaces:

- `apps/web`: marketing and sales
- `apps/portal`: customer operations
- `apps/admin`: internal operations
- `apps/docs`: developer onboarding
- `apps/api`: platform backend
- `apps/gateway`: public API routing layer
